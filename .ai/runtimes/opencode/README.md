# OpenCode Runtime Adapter

## Purpose
Add OpenCode as a peer runtime adapter that consumes canonical instructions from `AGENTS.md`, `INDEX.md`, and `.ai/*`.

## Canonical Rule
- Canonical source remains `AGENTS.md`, `INDEX.md`, and `.ai/*`.
- `.ai/runtimes/opencode/*`, `.opencode/commands/*`, and `opencode.json` are committed runtime adapter assets in this source repo.
- `.opencode/agents/*` are generated runtime artifacts for consumer projects.
- If adapter assets conflict with canonical files, canonical files win.

## Findings From Existing Runtime Patterns
- Runtime adapters are derivative integration layers, not parallel architecture.
- Claude/Codex keep orchestration in runtime main session and map canonical roles into runtime-native agent artifacts.
- Runtime docs emphasize mapping, drift avoidance, and governance preservation.
- Workflow execution and delegation rules are preserved from canonical `.ai/execution/*`, `.ai/delegation/*`, and `.ai/policies/*`.

## OpenCode Integration Surface
- Root config: `opencode.json`
- Runtime mapping docs: `.ai/runtimes/opencode/*`
- Runtime-native generated agents (consumer project output): `.opencode/agents/*.md`
- Runtime-native commands: `.opencode/commands/*.md`
- Runtime-managed dependency artifacts may be created by OpenCode under `.opencode/` (for example `.opencode/package.json`, `.opencode/package-lock.json`, `.opencode/node_modules/`).

## Artifact Ownership
- `.opencode/agents/*.md` are framework adapter outputs generated by `build-opencode-agents`.
- `.opencode/package.json`, `.opencode/package-lock.json`, `.opencode/node_modules/` are OpenCode runtime-managed artifacts, not framework adapter outputs.
- OpenCode runtime-managed dependency artifacts should not be committed.
- In the `ai-agents` framework source repository, `.opencode/agents/*` must not be generated or committed.
- Generate `.opencode/agents/*` only in consumer/test projects that import framework assets.

## Build Generation Safety
- `build-opencode-agents` is adapter generation only.
- It must not install plugins/dependencies or create package-manager artifacts.
- `.opencode/commands/build-opencode-agents.md` is a startup-safe committed entrypoint for natural invocation (`build opencode agents`).
- In consumer/test repositories, run generation immediately without extra confirmation prompts.
- Framework-source guard is mandatory:
  - refuse generation in `ai-agents` source repo by default,
  - allow only when user explicitly states `override: generate opencode agents in source repo` (unsafe/diagnostic-only).
- Source-repo detection uses strong indicators (framework README identity, repo directory name `ai-agents`, framework git remote, optional source marker file).
- Presence of framework assets in consumer repos (`.ai/runtimes/*`, `.ai/workflows/build-opencode-agents.md`, `examples/`) is not a source-repo indicator.
